This example demonstrates Sticky States ($stickyState) and Previous State ($previousState) It intentionally does not enable sticky state on Personnel, Inventory, and Customer tabs

Click the "Personnel" tab.
  • Drill down through the Personnel states until you are editing an employee record.
  • Start adding some notes to the notes field.
  • Click the "Modal 1" tab. The state transitions to the modal state. The from states' views (DOM and scope) are retained.
  • Navigate to the modals substate, if desired.
  • Close the modal. You are redirected back to the previously inactivate state with the views retained.
