Overview

Sticky States

A Sticky State is the root of a UI-Router state tree which can continue running even after it is "exited". The sticky state (and its children) have a different lifecycle than normal states. The DOM, $scope, and controller of a Sticky State (and all activated substates) are retained until one of two things happen:

  • The parent of the sticky state is exited
  • The parent of the sticky state is directly activated

If a sibling of the sticky state (or a child of a sibling) is activated, the sticky state tree will become "inactivated", until the rules above cause it to exit.

This feature is based on a "tabs" use case. The use case requires that the states for a tab in the application does not get exited when switching to one of the other tabs. The user should be able to switch back and forth between tabs without any interruption in their workflow. The URL should reflect only the state tree of the active tab.

For the tabs use case, Sticky States work best when also marked as Deep State Redirect
